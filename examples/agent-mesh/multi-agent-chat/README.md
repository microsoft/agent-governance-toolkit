# AgentMesh — Basic Multi-Agent Chat Example

A minimal end-to-end demo of [`agentmesh`](../../../agent-governance-python/agent-mesh/): two mock agents exchange messages through `AgentMeshClient`, with one message allowed by policy and one blocked. The audit trail and Merkle-chain integrity check round out the governance story.

## What This Shows

| Step | API used | Outcome |
| --- | --- | --- |
| Wire two governed agents | `AgentMeshClient(agent_id, capabilities, policy_yaml)` | Two mock agents with identity + audit + trust |
| Exchange a message | `bob.receive(alice, action, message)` (calls `client.execute_with_governance`) | Returns `(GovernanceResult, reply_text)` |
| Block by policy | YAML rule with `action: deny` on `action.type == 'export'` | `allowed=False`, trust score drops, no reply |
| Verify audit chain | `client.audit_log.verify_integrity()` | Merkle-chain integrity confirmed |

## Prerequisites

- Python 3.11+
- pip

No API keys are required. After installation, the example runs locally and does not call external services.

## Quick Start

```bash
cd examples/agent-mesh/multi-agent-chat
python -m venv .venv

# Activate the venv — pick the right line for your shell:
#   Windows PowerShell:        .venv\Scripts\Activate.ps1
#   macOS / Linux / Git Bash:  source .venv/bin/activate

pip install -r requirements.txt
python main.py
```

## Expected Output

```text
AgentMesh Basic Multi-Agent Chat
============================================================
  alice-agent  did=<did>...  trust=500.0
  bob-agent    did=<did>...  trust=500.0

  [ok]      alice-agent -> bob-agent summarize  -> decision=allow trust=510.0
             reply: summary: governance policy allows this request
  [blocked] alice-agent -> bob-agent export     -> decision=deny  trust=490.0
             reply: blocked before processing

Bob's audit trail:
  - event=policy_evaluation  action=summarize  outcome=success  policy=allow
  - event=policy_evaluation  action=export     outcome=denied   policy=deny

Audit integrity: verified
```

Notes on what to expect run-to-run:

- **Trust scores** are deterministic with the default deltas: `500.0` initial, `+10` on allow → `510.0`, `-20` on deny → `490.0`.
- **Decision strings** (`allow`, `deny`) come straight from the matched policy rule's action.
- **Agent DIDs** are generated by `AgentIdentity.create()` from a fresh keypair and random seed material, so they differ every run — shown above as `<did>` for portability.
- **Audit trail order and content** are deterministic; both events are `policy_evaluation` rows, one per message.
- **Audit integrity** is `verified` because the Merkle-chained `AuditLog` validates every entry's hash linkage.

## How It Maps to the SDK

- **Unified governance pipeline** → [`agentmesh.AgentMeshClient`](../../../agent-governance-python/agent-mesh/src/agentmesh/client.py), [`agentmesh.GovernanceResult`](../../../agent-governance-python/agent-mesh/src/agentmesh/client.py)
- **Policy enforcement (under the hood)** → [`agentmesh.governance.policy.PolicyEngine`](../../../agent-governance-python/agent-mesh/src/agentmesh/governance/policy.py), `Policy`, `PolicyRule`, `PolicyDecision`
- **Audit logging (under the hood)** → [`agentmesh.governance.audit.AuditLog`](../../../agent-governance-python/agent-mesh/src/agentmesh/governance/audit.py), `AuditEntry`
- **Identity (under the hood)** → [`agentmesh.identity.AgentIdentity`](../../../agent-governance-python/agent-mesh/src/agentmesh/identity/agent_id.py)
- **Trust scoring (under the hood)** → [`agentmesh.reward.scoring.TrustScore`](../../../agent-governance-python/agent-mesh/src/agentmesh/reward/scoring.py)

For the full SDK overview, see [`agent-governance-python/agent-mesh/README.md`](../../../agent-governance-python/agent-mesh/README.md).
