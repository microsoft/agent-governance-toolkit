## Security advisory relevance

vitest 4.1.0 → 4.1.8 includes fixes for advisories surfaced by OSSF Scorecard, all in the test runner toolchain (dev-only). None of the vulnerable code paths execute at runtime in shipped packages because vitest is in `devDependencies` and excluded from npm publish via the existing `files` allowlist in each `package.json`.

No CVE filings reference our published packages. Confirmed via `npm audit --production` (zero high/critical in runtime tree).

## Breaking change risk

**Risk: low.** vitest 4.1.x is the same minor; patch bumps within 4.1 are non-breaking per vitest's release policy. Local `npm test` in both workspaces passes against the new lockfile. CI matrix re-runs vitest for both packages and any incompatibility would surface there.

## Rollback plan

Revert `package.json` + `package-lock.json` in both directories to `vitest@4.1.0` and re-run `npm install --legacy-peer-deps`.
