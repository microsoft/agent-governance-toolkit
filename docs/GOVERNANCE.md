# Governance

This page summarizes AGT governance for documentation readers. The source of
truth is the root [GOVERNANCE.md](../GOVERNANCE.md), [MAINTAINERS.md](../MAINTAINERS.md),
and [OWNERS.md](../OWNERS.md).

AGT is proposed for AAIF hosting in `aaif/project-proposals#19`; governance is
being aligned with LF/AAIF project policy during contribution finalization.

## Decision making

Major technical and governance changes are discussed in public issues, ADRs, or
pull requests. Normative specification changes follow the
[Specification Change Process](specs/PROCESS.md).

## Releases

Canonical releases are approved by release managers listed in [OWNERS.md](../OWNERS.md)
and follow [docs/RELEASE.md](RELEASE.md). Microsoft ESRP is not a canonical AGT
release path.

## Code of Conduct and security

The project follows the root [CODE_OF_CONDUCT.md](../CODE_OF_CONDUCT.md) and
[SECURITY.md](../SECURITY.md). Do not report vulnerabilities through public
issues.
